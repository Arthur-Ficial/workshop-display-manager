Symptom: clicking rotation segments (0°/90°/180°/270°), flip segments (—/H/V), and ACTIONS rows in the Inspector has perceptible lag — feels closer to several hundred ms than instant. Workshop facilitators expect a native-Mac-app feel. Investigate, in order:
    1. AX walker `snapshot()` cost — `AXUIElementCreateApplication` + recursive `AXUIElementCopyAttributeValue` runs each /ui/snapshot. Profile to see if it dominates. Cache by AX version when stable; only re-walk on AppKit notifications.
    2. `AXUIElementPerformAction` is synchronous; the click handler may block the main thread on each press — measure wall time on a real headed run and confirm.
    3. Combine `combineLatest` rebuilds the whole RemoteRegistry on every tile/selected/profiles change — diff instead of replace, or debounce.
    4. SwiftUI re-renders on every `@Published` write — check whether the InspectorGeometry / InspectorActions VStacks are doing more re-layout than necessary; consider equatable-bound child views.
    5. WebKit Stage `setState(...)` JSON encode + JS evaluation per change — already async but check the round-trip cost.
  - Acceptance: clicking any Inspector control yields visible feedback within 50 ms and the corresponding state change within 100 ms. CLAUDE.md says no fakes — measure with `os_signpost` or `tracy`-style timing, not vibes.
  - Verify: a real-hardware micro-benchmark, plus a hermetic e2e that asserts `/ui/click` → `/ui/snapshot` round-trip is under N ms on the test rig.
  - Scope: M-L (depends on which culprit dominates).
