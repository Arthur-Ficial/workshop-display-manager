### #B-1 PIP-CURSOR-WARP ‚Äî clicking PIP teleports cursor to source display
- Reported: 2026-05-09 by user testing /Applications/WDMMac.app PiP feature
- Root cause: GUI `vm.openPiP` calls `controller.pip(plan:using:)` with the
  default `PipPlan(remoteControl: false)` ‚Äî but the user reports the cursor
  IS warping. Investigation: `RemoteControlPipView` warps on every event,
  but it should only be installed when `remoteControl == true`. The plain
  PiP path uses a passive `NSView`. Verify the GUI is on the correct branch
  and add a regression test.
- Fix scope: ensure `vm.openPiP` always uses `remoteControl: false` (workshop
  facilitators want a passive mirror, not a remote-control surface).
  Regression test: spawn the GUI under `RecordingPipFlipper`, click Open PiP,
  assert the recorded log contains `remoteControl=false`.

### #B-2 PIP-ASPECT-RATIO ‚Äî resizing PIP window distorts the picture
- Reported: 2026-05-09 by user testing /Applications/WDMMac.app PiP feature
- Root cause: `AppKitPipFlipper` creates `NSWindow` without
  `contentAspectRatio`, so dragging a corner stretches the captured frame
  into a non-source ratio.
- Fix scope: set `win.contentAspectRatio = NSSize(W,H)` from the source
  display's pixel dims before `makeKeyAndOrderFront`. Regression test:
  spawn PiP via fixture, set window size 800x300, assert the resulting
  contentAspectRatio matches the source display's W:H ratio.

### #B-3 STAGE-WALLPAPER-STALE ‚Äî Stage tile preview shows old wallpaper
  even after `wdm wallpaper set` / "Change background" replaces it
- Reported: 2026-05-09 by user (screenshots: tiles still show beach when
  every desktop is now solid blue).
- Root cause: `Tile.wallpaperURL` is read once per `reload()`. The wallpaper
  POLL interval may not be observing background changes (no FSEvent on the
  cached URL or the URL itself doesn't change when the user picks a new
  solid color). VM needs to invalidate the cached image when wallpaper URL
  changes OR poll wallpaper at a sane cadence.
- Fix scope: ensure `vm.reload()` re-reads wallpaper URLs every tick, and
  that the Stage tile renders the URL at the time of stamp. Regression
  test: change wallpaper URL on fixture, call `vm.reload()`, assert the
  new URL is on the tile. Visual: change Mac wallpaper, watch Stage update
  within 1s.

### #B-4 STAGE-AUTOFIT ‚Äî Stage canvas should auto-fit + center all
  monitors so every tile is always visible (no manual pan needed)
- Reported: 2026-05-09 by user (screenshots: tiles clipped to bottom-right
  of canvas; user wants zoom + offset auto-computed). TDD requested.
- Fix scope: compute the smallest zoom + canvas offset that fits the
  union of every tile's `(originX, originY, widthPx, heightPx)` rect into
  the visible canvas with a small inset. Re-fit whenever the tile set
  changes (display added/removed/resized). Regression tests in
  `WDMCoreTests/StageAutoFitTests.swift` (pure math): given a list of
  tile rects + a canvas size, return zoom + offset such that every tile
  is inside the canvas and the bounding box is centered.
